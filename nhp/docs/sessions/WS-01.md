## Goal
Stand up the NHP app: a UI ported from `rubix-old/ui` stripped to an app shell (auth + nav +
layout + API client) pointed at the **already-built** rubix backend, plus a `nhp/Makefile` copied
from `rubix/Makefile`. This is the root every later WS builds on — no NHP features yet, just a
running `make dev` that serves a shell and talks to the backend.

## Dependencies
None. First WS. The rubix backend exists and runs (see [../OVERVIEW.md](../OVERVIEW.md)).

## Standard
[rubix/docs/FILE-LAYOUT.md](../../../rubix/docs/FILE-LAYOUT.md) — one responsibility per file,
verb-per-file folders, ≤400 lines, no utils/helpers/common. Barrels (`index.ts`) re-export only.

## Current state (re-verify first)
- `rubix-old/ui` is a Vite + React + TanStack + shadcn app (`package.json` name `shadcn-admin`) with
  `src/features/{admin,tenants,builder,points,settings,time,auth,nav,…}`. Rich parts bin — copy and
  strip, don't rebuild.
- `rubix/ui` (name `rubix-ui`) is the newer Laminar UI with `src/api/{boards,charts}.ts`, the
  chart-builder, and the dashboards components referenced by
  [rubix DASHBOARDS-SCOPE](../../../rubix/docs/design/DASHBOARDS-SCOPE.md). Prefer its API-client
  patterns where they're cleaner; prefer rubix-old's feature scaffolding for admin/nav.
- `rubix/Makefile` already does build/dev/dev-be/dev-ui/test/lint/fmt/db-up/kill with `BE_PORT 8092`,
  `UI_PORT 5192`, `RUBIX_BIND`, `VITE_API_PROXY`. JS toolchain is off PATH (nvm); cargo under
  `~/.cargo/bin` — the Makefile resolves these; keep that.

## Tasks (incremental — commit per task, `WS-01:` prefix)
1. Copy `rubix-old/ui` → `nhp/ui`. Update `package.json` name to `nhp-ui`. Get `pnpm install` +
   `pnpm build` green as-is before stripping anything.
2. Strip to a shell: keep layout, nav, theme, auth, the API client, and the command menu; remove
   feature pages that are clearly rubix-domain and not NHP (note removals in the session log). Don't
   delete shared UI primitives (`components/ui`).
3. Point the API client at the rubix backend: base URL via `VITE_API_PROXY`; wire auth against the
   rubix principal/session endpoints (`/principals`, the auth/session surface — re-grep
   `rubix/crates/rubix-server/src/http`). A login that yields a scoped session is enough for the POC.
4. Add an NHP nav scaffold with placeholder routes for the coming features (Dashboards, Admin,
   Wizards) — empty pages are fine; later WSs fill them.
5. Copy `rubix/Makefile` → `nhp/Makefile`; adapt `UI_DIR`, ports (pick non-clashing values if NHP
   and rubix may run together, e.g. `BE_PORT ?= 8094`, `UI_PORT ?= 5194`), and the dev target so
   `make dev` runs the rubix backend + the NHP UI together. Document the chosen ports in the session
   log. See [../SEED.md](../SEED.md).

## Done gate
- `pnpm -C nhp/ui build` succeeds; `pnpm -C nhp/ui dev` serves the shell.
- `make -C nhp dev` boots the rubix backend + NHP UI and the UI can reach the API (a login or a
  `/health` ping succeeds through the proxy).
- No TypeScript errors; no `@ts-ignore` added to dodge them.
