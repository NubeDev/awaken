## Goal
Stand up the NHP app: a UI ported from `rubix-old/ui` stripped to an app shell (auth + nav +
layout + API client) pointed at the **already-built** rubix backend, plus a `nhp/Makefile` copied
from `rubix/Makefile`. This is the root every later WS builds on — no NHP features yet, just a
running `make dev` that serves a shell and talks to the backend.

## Dependencies
None. First WS. The rubix backend exists and runs (see [../OVERVIEW.md](../OVERVIEW.md)).

## Standard
[rubix/docs/FILE-LAYOUT.md](../../../rubix/docs/FILE-LAYOUT.md) — one responsibility per file,
verb-per-file folders, ≤400 lines, no utils/helpers/common. Barrels (`index.ts`) re-export only.

## Current state (re-verify first)
- `rubix-old/ui` is a Vite + React + TanStack + shadcn app (`package.json` name `shadcn-admin`) with
  `src/features/{admin,tenants,builder,points,settings,time,auth,nav,…}`. Rich parts bin — copy and
  strip, don't rebuild.
- `rubix/ui` (name `rubix-ui`) is the newer Laminar UI with `src/api/{boards,charts}.ts`, the
  chart-builder, and the dashboards components referenced by
  [rubix DASHBOARDS-SCOPE](../../../rubix/docs/design/DASHBOARDS-SCOPE.md). Prefer its API-client
  patterns where they're cleaner; prefer rubix-old's feature scaffolding for admin/nav.
- `rubix/Makefile` already does build/dev/dev-be/dev-ui/test/lint/fmt/db-up/kill with `BE_PORT 8092`,
  `UI_PORT 5192`, `RUBIX_BIND`, `VITE_API_PROXY`. JS toolchain is off PATH (nvm); cargo under
  `~/.cargo/bin` — the Makefile resolves these; keep that.

## Tasks (incremental — commit per task, `WS-01:` prefix)
1. Copy `rubix-old/ui` → `nhp/ui`. Update `package.json` name to `nhp-ui`. Get `pnpm install` +
   `pnpm build` green as-is before stripping anything.
2. Strip to a shell: keep layout, nav, theme, auth, the API client, and the command menu; remove
   feature pages that are clearly rubix-domain and not NHP (note removals in the session log). Don't
   delete shared UI primitives (`components/ui`).
3. Point the API client at the rubix backend: base URL via `VITE_API_PROXY`; wire auth against the
   rubix principal/session endpoints (`/principals`, the auth/session surface — re-grep
   `rubix/crates/rubix-server/src/http`). A login that yields a scoped session is enough for the POC.

   Wizards) — empty pages are fine; later WSs fill them.
5. Copy `rubix/Makefile` → `nhp/Makefile`; adapt `UI_DIR`, ports (pick non-clashing values if NHP
   and rubix may run together, e.g. `BE_PORT ?= 8094`, `UI_PORT ?= 5194`), and the dev target so
   `make dev` runs the rubix backend + the NHP UI together. Document the chosen ports in the session
   log. See [../SEED.md](../SEED.md).

Ports chosen: **BE_PORT 8094 / UI_PORT 5194** (coexist with rubix 8092/5192 and the 8088/5180
defaults). The NHP UI is self-contained: an empty `pnpm-workspace.yaml` (mirrors rubix/ui) stops
pnpm walking up to the repo-root workspace.

Commits (each `WS-01:` prefixed):
1. **copy rubix-old/ui → nhp/ui as nhp-ui, build green** — verbatim port, renamed package to
   `nhp-ui`, added the self-contained workspace file. `pnpm install` + `pnpm build` pass unmodified.
2. **strip nhp/ui to NHP app shell** — kept layout/theme/auth/api-client/ui-primitives; replaced
   the org/site-scoped nav with a flat static NHP nav (Dashboards/Admin/Wizards) + placeholder
   routes (`/dashboards`, `/admin`, `/wizards`) via a shared `PlaceholderPage`; root lands on
   `/dashboards`. Removed the rubix-domain `o/$org` + `settings` route subtrees and their features.
3. **add nhp/Makefile + point UI dev proxy at rubix routes** — Makefile runs cargo in `../rubix`
   (NHP has no backend crate); vite proxies the rubix flat routes to `VITE_API_PROXY`.

Auth: rubix-old already ships a POC-grade token-paste sign-in (`/sign-in`) that sends a bearer
token on the rubix `/api/v1/*`/flat routes, and main.tsx redirects to `/sign-in` on a 401. That is
the "login that yields a scoped session" the spec accepts — no new auth code was needed. The API
client (`api/client.ts`) was already pointed at the backend via a same-origin base + the vite proxy.

Deviations (all to keep the shell building and sever rubix-domain surface):
- **Dropped the command menu / SearchProvider.** WS-01 lists "keep the command menu", but rubix-old's
  command-menu is hard-wired to the `ask-awaken` AI feature (out of NHP scope). Severing ask-awaken
  cleanly meant removing the command menu too. A later WS can reintroduce a plain command palette.
- **Dropped ScopeProvider (org/site URL scope).** NHP's nav is flat (records browsed inside feature
  pages, not encoded in the path), so the whole `o/$org` scope system was removed.
- **Deleted the rubix-domain api hooks/endpoints/stream monolith** (`api/hooks.ts` was 1075 lines,
  itself over the 400-line ceiling and referencing the deleted features). Kept `api/client.ts`
  (the fetch wrapper) and `api/types.ts`. WS-02 defines the NHP endpoint surface.
- **PreferencesProvider now serves baseline defaults** instead of fetching `GET /me/preferences` —
  that endpoint isn't wired on rubix (OVERVIEW gap #3), and removing the fetch severed the api
  monolith. A later WS can restore the fetch once the prefs endpoint exists.
