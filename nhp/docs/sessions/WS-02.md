## Goal
Land the NHP domain as **rubix collection records** (`kind:"collection"`) — tenant, site, gateway,
network, meter, meter-type, register — with typed fields, `required`/`unique` rules, relation links,
and write-rules (incl. the per-network device limit). Resolve the closed-enum strategy. After this
WS, creating an NHP record validates against its shape; no later WS has to define the schema.

## Dependencies
WS-01 (committed). The rubix collection layer is built: `kind:"collection"` records, `FieldType
{Text, Number, Bool, Date, File, Relation}`, gate write-path validation (fail-open / strict), and
`required`/`unique` — verified in [../OVERVIEW.md](../OVERVIEW.md).

## Standard
[rubix/docs/FILE-LAYOUT.md](../../../rubix/docs/FILE-LAYOUT.md). Collection definitions are data;
if authored as code (a seed-style registrar), one file per collection.

## Current state (re-verify first)
- Collection model + field types: `rubix/crates/rubix-core/src/collection/` (`def.rs`, `field.rs`,
  `bootstrap.rs`). Validation: `rubix/crates/rubix-gate/src/command/validate.rs`.
- **No `Select`/enum field type** (OVERVIEW gap #1). NHP enums: `network.net_type` (485/ethernet),
  `network.protocol` (modbus), register `datatype`/`fn_code`/`byte_order`, `chart_type`.

## Tasks (incremental — commit per task, `WS-02:` prefix)
1. Author the 7 collection definitions per [../DOMAIN-MODEL.md](../DOMAIN-MODEL.md) — every field,
   type, `required`, `unique`, and relation. Register them through the gate (a small registrar
   module or a seed step that upserts the `kind:"collection"` records). One file per collection.
2. **Enum strategy — NHP-only, NO rubix change.** Model every closed enum as a `text` field and
   enforce the allowed set with data that lives ON the collection record + the UI form:
   - the collection `writeRule` (an access/validation expression — *data*, not core code) rejecting
     a value outside the set, and/or
   - the UI dropdown (WS-04/05) constraining input.
   **Do NOT modify rubix core.** Adding a `Select`/enum `FieldType` to
   `rubix/crates/rubix-core/src/collection/field.rs` is explicitly OUT OF SCOPE — NHP runs on the
   already-built rubix backend unchanged. If rubix's `writeRule` mechanism turns out to be unable to
   enforce an allowed-set (re-grep the validate path to confirm), that is a **TODOs.md blocker**
   (and the UI dropdown carries enforcement for the POC) — it is NOT a license to patch rubix.
3. Add the device-limit `writeRule` on `meter`: reject a create whose parent `network` already has
   `max_devices` meters (defence-in-depth behind the wizard's check). If write-rules can't express a
   count query, log to TODOs.md and rely on the wizard check for the POC (note the gap).
4. Verify validation end-to-end: a well-formed record of each kind writes; a bad one (missing
   required, bad enum, dup unique) is rejected with a clear error.

## Done gate
- All 7 collections register cleanly against the **unmodified** rubix backend (idempotent re-run).
- A scripted check creates one valid record per kind (writes) and one invalid per kind (rejected).
- **No rubix source is changed** — `git status` shows zero diff under `rubix/` (the collections are
  data + a registrar that lives under `nhp/`). If `rubix/` shows changes, the gate FAILS.

## Notes / follow-ups

- Stamp-on-create + `meter_type_version` (meter-type versioning, DOMAIN-MODEL §versioning) is set up
