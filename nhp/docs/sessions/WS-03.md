Status: Pending

## Goal
A one-command seed that populates a realistic NHP portfolio through the gate, plus a stand-in for
the (not-built) polling service: fake `status`/`last_seen` and back-filled `history` for
`history=true` registers, so every UI workstream has live-looking data to render.

## Dependencies
WS-02 (collections committed). Modelled on rubix's seed
(`rubix/crates/rubix-server/src/seed/{portfolio,history,cast}.rs`) — same gate-written pattern.

## Standard
[rubix/docs/FILE-LAYOUT.md](../../../rubix/docs/FILE-LAYOUT.md). Verb/spec-per-file: portfolio
builder, history generator, status faker as separate files. No one mega seed file.

## Current state (re-verify first)
- rubix seed: `--seed-dev` arg in `rubix/crates/rubix-server/src/main.rs` calls `seed_dev`; the
  portfolio is built record-by-record **through the gate** so it carries real audit/undo/events.
  Reuse `seed/history.rs`'s series generator for the back-fill.
- Seed shape to build is specified in [../SEED.md](../SEED.md).

## Tasks (incremental — commit per task, `WS-03:` prefix)
1. **Meter-types:** 1–2 full types with real Modbus register maps (3-phase: V L1/L2/L3, A L1/L2/L3,
   kW, kWh, Hz, PF) — each register with address/datatype/scale/byte_order/unit/quantity/history/
   chart_type/chart_group/alarm per [../DOMAIN-MODEL.md](../DOMAIN-MODEL.md).
2. **Portfolio:** 2 tenants × 2 sites; per site 1–2 gateways, each with a mix of 485 + ethernet
   networks (with `max_devices` caps), and several meters stamped from the types. Apply the standard
   tags (`tenant:…`, `site:…`, `gateway:…`, `meter:…`, `group:…`, `quantity:…`) so dashboards
   auto-build (DASHBOARDS.md).
3. **Users/teams:** admin + operator + viewer per tenant, and a **poller service-account** principal
   with write grants — per [../ADMIN.md](../ADMIN.md) roles.
4. **Fake the poller:** write plausible `status`/`last_seen`; back-fill history for `history=true`
   registers (reuse rubix `seed/history.rs`); a ticker for live values during `make dev SEED=1`.
   This is the ONLY place NHP fabricates device data — it stands in for the external service.
5. Wire `SEED=1` into `nhp/Makefile` (mirror rubix's `SEED_FLAG`), off by default.

## Done gate
- `make -C nhp dev SEED=1` populates the portfolio and the seed run exits clean.
- A scripted check (or query console) confirms expected counts: 2 tenants, 4 sites, ≥1 meter-type
  with a full register set, meters tagged, history present for `history=true` registers.
- Seed is idempotent-ish (repopulates on boot, documented), off by default.

## Notes / follow-ups
- Write through the gate as the tenant operator — do NOT bypass with direct store writes (keeps
  audit/undo/events real, which the admin/dashboard WSs rely on).
