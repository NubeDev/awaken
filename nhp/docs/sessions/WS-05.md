## Goal
The infrastructure + identity admin: CRUD for gateways and their networks (485/ethernet, protocol,
per-network `max_devices` device limit), and user/team/role management on the rubix principal +
grant surface. Covers [../ADMIN.md](../ADMIN.md) §5–6.

## Dependencies
WS-02 (collections), WS-03 (data), WS-01 (shell). Independent of WS-04 (different collections) but
runs after it per the one-branch sequential rule.

## Standard
[rubix/docs/FILE-LAYOUT.md](../../../rubix/docs/FILE-LAYOUT.md). One file per verb/route. Reuse
rubix-old/ui `features/admin/{members,teams,access}.tsx` for users/teams/roles.

## Current state (re-verify first)
- rubix admin API is built: `/principals` (+ `/principals/:subject/grants`), `/tenants`, `/devices`
  — `rubix/crates/rubix-server/src/http/admin/`. Roles model: role-in-namespace
  ([rubix ADMIN-API.md](../../../rubix/docs/design/ADMIN-API.md)).
- NHP role set (viewer/operator/admin/root) is in [../ADMIN.md](../ADMIN.md) §6.

## Tasks (incremental — commit per task, `WS-05:` prefix)
1. **Gateways:** list/create/edit/delete (`gateway` collection); show online/offline + `last_seen`
   (poller-written, read-only here).
2. **Networks:** under a gateway, CRUD `network` records — `net_type` (485/ethernet, switches the
   `params` form: baud/parity vs ip/port), `protocol`, `max_devices`. Show device count vs cap.
   Unlimited networks per gateway.
3. **Device-limit enforcement:** the create-network/add-meter UI shows remaining capacity and blocks
   over-cap; rely on WS-02's `writeRule` as the backend backstop (or note its gap if write-rules
   can't count).
4. **Users & teams:** CRUD principals, assign NHP **roles** (viewer/operator/admin) and team
   membership via `/principals` + grants. Surface the poller service-account as a managed principal.
