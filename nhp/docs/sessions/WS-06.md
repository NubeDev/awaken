## Goal
Guided multi-step wizards that author many gate-audited records (and their tags) at once, applying
NHP conventions so dashboards auto-build. Covers [../WIZARDS.md](../WIZARDS.md): new tenant, new
site, new gateway **with N networks** ("30 networks"), bulk meters, new user/team, and the combined
"add everything" flow.

## Dependencies
WS-04 + WS-05 (the create paths and the device-limit rule the wizards orchestrate), WS-02/03/01.

## Standard
[rubix/docs/FILE-LAYOUT.md](../../../rubix/docs/FILE-LAYOUT.md). One file per wizard, one file per
step within a wizard (`gateway-wizard/{gateway-step,networks-step,preview-step}.tsx`). No mega
wizard file.

## Current state (re-verify first)
- Wizard flows + behaviours specified in [../WIZARDS.md](../WIZARDS.md). Each step is an ordinary
  collection write (WS-04/05 paths). rubix-old/ui form/stepper primitives are the parts bin.

## Tasks (incremental — commit per task, `WS-06:` prefix)
1. **Wizard shell:** a reusable stepper with a final **preview** of records-to-create, then a
   batched write with per-record results; resumable on partial failure (don't restart from zero).
2. **Gateway + N networks** (the headline): one step generates N network records from a count, a
   `net_type`/`protocol`/`max_devices`, a naming pattern (`gw-01-net-{n}`), and per-type `params`
   defaults. Preview all N, confirm, write. Test with N=30.
3. **Bulk meters onto a network:** pick network (shows remaining = `max_devices` − current), pick a
   meter-type, add a range of bus addresses; **block** over-cap; stamp registers + tags per meter.
4. **Tenant / site / user wizards:** tenant (root, fresh namespace via `/tenants` + first admin),
   site (name/address/timezone), user/team (role assignment).
5. **Combined flow:** chain tenant→site→gateway(+networks)→meters→users, threading parent ids, one
   final preview of the whole tree.

## Done gate
- `pnpm -C nhp/ui build` green, no TS errors.
- Smoke: run the gateway+30-networks wizard → 30 networks created + tagged; run bulk-meters →
  capped correctly, meters stamped; run the combined flow on a fresh tenant → dashboards (WS-07)
  show the new tree.

## Notes / follow-ups
- Wizards apply the standard tags (WIZARDS.md) — keep tag construction in one shared module so it
  matches WS-03's seed and WS-07's auto-build exactly (a drift here silently breaks auto-build).
