## Goal
Wrap the POC into something demoable: a scripted end-to-end path (seed → wizard → admin → dashboard),
an `nhp/README.md` that says how to run it, and fixes for whatever the smoke surfaces. The "it
actually works front to back" workstream.

## Dependencies
Everything (WS-01–07). Runs last.

## Standard
[rubix/docs/FILE-LAYOUT.md](../../../rubix/docs/FILE-LAYOUT.md).

## Tasks (incremental — commit per task, `WS-08:` prefix)
1. **End-to-end smoke:** a script (or documented manual path) that: boots `make dev SEED=1`, logs in,
   runs the gateway+networks wizard, adds a meter-type + meters, opens the auto-built dashboard, sees
   history + an alarm. Capture each step's expected result.
2. **Fix what breaks:** anything the smoke surfaces that's within an existing WS's lane — fix in
   place (small, `WS-08:` prefixed). Anything larger → TODOs.md, don't paper over it.
3. **README:** `nhp/README.md` — what NHP is (one paragraph, link OVERVIEW), how to run
   (`make dev SEED=1`, ports, login), the demo walkthrough, and the known POC gaps (OVERVIEW gaps +
   anything in TODOs.md).
4. **Demo walkthrough:** a short `nhp/docs/DEMO.md` — the click-path that shows the product working,
   for showing stakeholders.

## Done gate
- The smoke path runs clean front to back on a fresh checkout (`make dev SEED=1`).
- `pnpm -C nhp/ui build` green.
- README + DEMO committed; TODOs.md reflects any remaining known gaps honestly.

## Notes / follow-ups
- This is the POC finish line. If the smoke reveals a missing backend capability beyond the OVERVIEW
  gaps, that's a TODOs.md entry + a README "known limitation", not an unscoped backend build.
