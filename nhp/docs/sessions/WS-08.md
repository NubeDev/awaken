## Outcome (2026-06-16)

### What the smoke covers
`make smoke` (→ `scripts/smoke.sh`) is the single entrypoint, wiring the EXISTING checks rather than
inventing new ones. It owns its own backend — needs nothing running first — and runs 7 gated steps,
each aborting non-zero on failure (`set -euo pipefail`):
1. Boot a THROWAWAY `rubix --seed-dev` with `RUBIX_DATA_DIR` pointed inside `rubix/` (gitignored
   `/rubix-data`), killed + `rm -rf`'d on exit (trap) so the worktree root never grows a stray
   `rubix-data/` and rubix's tree stays clean. Polls `/records?kind=tenant` with the seeded
   `acme_operator` creds until ready (90s budget).
2. `register-collections.mjs` — the 7 NHP collection definitions.
3. `seed/seed.mjs` — mock portfolio + faked poller data.
4. `seed/check.mjs` (`seed-check`) — portfolio counts present.
5. `seed/records-check.mjs` (NEW, WS-08) — the headless equivalent of "open a dashboard": asserts
   the dashboard data pipeline (`ui/.../dashboards/query/batch.ts` reads each kind whole over
   `/records`) returns rows for tenant/site/gateway/meter/register/history, that every gateway
   carries `status` and at least one is offline (status rollup → degraded), and that history joins to
   a meter for a trend chart.
6. `pnpm install --frozen-lockfile && pnpm build`.
7. `pnpm test:unit`.

### Run result (actually executed, unsandboxed, against a fresh `--seed-dev` backend)
**SMOKE PASSED front to back.** Backend ready at ~11s; collections registered; seed wrote the
portfolio; `seed-check` all PASS; `records-check` all PASS (dashboard `/records` pipeline returns
rows + offline gateway present + history→meter join); `pnpm build` green (2746 modules, vite OK);
`pnpm test:unit` **55/55 passed (11 files)**. Final line: `SMOKE PASSED — boot → collections → seed →
seed-check → records-check → build → unit tests`.

### What I fixed
- Nothing was broken in WS-01..07; the smoke surfaced no in-lane defect to fix. The only thing the
  first run surfaced was an ENVIRONMENT constraint: the agent's sandboxed shell blocks localhost
  networking, so node `fetch`/`curl` to `127.0.0.1:8094` failed with "fetch failed" / http=000 even
  though the server bound fine. Re-run with the sandbox disabled → all 7 steps pass. This is a harness
  constraint, not a POC bug — the script is correct. Also bumped the boot-wait from 60→90s for
  cold-boot safety.
