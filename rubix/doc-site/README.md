# Rubix Docs (`@rubix/doc-site`)

Production documentation site for **Rubix**, built with
[Nextra 4](https://nextra.site).

> Internal architecture specs live in `rubix/docs/design` and are **not**
> published. This site is the user-facing, curated documentation.

## Develop

```sh
pnpm install
pnpm --filter @rubix/doc-site dev
# http://localhost:3010
```

## Build (static export)

```sh
pnpm --filter @rubix/doc-site build
# output: doc-site/out
```

The build statically exports to `out/` (no server at runtime). Configure these
env vars as needed:

- `DOCS_REPO_URL` — source repo for the navbar / "Edit this page" links
  (default: a neutral placeholder).
- `DOCS_BASE_PATH` — sub-path the site is served from, when not at the host
  root (e.g. `/rubix`). Empty locally.

## Authoring

- Pages are MDX under `content/`.
- Sidebar order / labels come from `_meta.js` files in each folder.
- The **Changelog** page is generated from `../CHANGELOG.md` by
  `scripts/sync-changelog.mjs` (runs on `predev` / `prebuild`). Edit the
  changelog there, never `content/changelog.mdx`.

## Versioning

See [VERSIONING.md](./VERSIONING.md). Today the site is **latest-only**;
per-release snapshots come later via git tags.

## Search

Search uses **Pagefind**, which indexes the built `.html`. Pagefind has nothing
to index until a static build exists, so historically the `pnpm dev` search box
showed "Failed to load search index".

**Search now works in `pnpm dev`.** The `predev` step
(`scripts/build-search-index.mjs`) indexes the last static build and copies the
index into `public/_pagefind`, which `next dev` serves at `/_pagefind/…` — the
path Nextra fetches. Notes:

- It does **not** trigger a build. If there's no `out/` yet (fresh checkout), it
  prints a hint and skips — run `pnpm build` once, then `pnpm dev`.
- The dev index reflects the **last build**, so newly-edited content won't be
  searchable until you re-run `pnpm build` (or restart `pnpm dev` after a build).
  Live page content still hot-reloads as usual; only the search index is stale.
- `public/_pagefind` is generated (git-ignored). Delete it freely.

The production index is generated by the `postbuild` step
(`pagefind --site out --output-path out/_pagefind`). The explicit `--output-path`
matters: Pagefind ≥1.x defaults to `out/pagefind/`, but Nextra fetches
`/_pagefind/pagefind.js`, so the output must be `out/_pagefind`.

## Troubleshooting

- **`Cannot find module './NNN.js'` from `.next/server/...` in dev** — the
  `.next` cache got clobbered, usually by running `pnpm build` while
  `pnpm dev` was live (both write `.next/`). Stop dev, `rm -rf .next` (or
  `make docs-clean`), then restart dev. Don't run a production build against the
  same checkout while the dev server is running.
- **`GET /favicon.ico 500 … missing param "/favicon.ico" in generateStaticParams()`**
  — the browser auto-requests `/favicon.ico`; with no icon present it fell
  through to the `[[...mdxPath]]` catch-all, which can't serve it under
  `output: export`. Fixed by `app/favicon.ico` (App-Router metadata file route),
  which takes precedence over the catch-all and is exported automatically.
- **Dev console: `Invalid prop \`ref\` supplied to \`React.Fragment\``** — emitted
  from inside `nextra/dist/client/components/search.js` (Nextra's Search passes a
  `ref` through a headlessui Combobox `Fragment`). Upstream, dev-only, harmless;
  no fix on current `nextra@4.6.1`. Not in our code — ignore it.
- **Dev console: hydration mismatch citing `cz-shortcut-listen` (or similar) on
  `<body>`** — injected by a browser extension (ColorZilla, Grammarly, …) before
  React hydrates. `<body suppressHydrationWarning>` in `app/layout.jsx` silences it.

## Pinned dependencies (why)

- `next` is pinned to `15.3.5` — Next 15.5's RSC bundler breaks nextra's static
  export ("Could not find module ... in the React Client Manifest").
- `nextra`'s `zod` is pinned to `4.1.12` (via root `pnpm.overrides`) — zod ≥4.4
  makes `z.custom()` reject `undefined`, crashing every page render.
