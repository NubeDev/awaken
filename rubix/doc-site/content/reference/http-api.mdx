# HTTP API

Rubix serves a JSON HTTP API. Every **mutation** crosses the access gate and is
audited; **reads** run on a scoped SurrealDB session. The server generates an
OpenAPI document — treat that as the source of truth and generate this page from
it rather than hand-maintaining endpoint lists.

## Surfaces (high level)

- **Records / collections** — typed CRUD over the generic record store
  (`/records`, `/ws/records`), with `kind`/tag listing and filtered realtime.
- **Readings** — append-only time-series append and windowed reads
  (`/readings`).
- **Query** — run queries across registered datasources, batched and streaming,
  with variable templating and chart rendering (`/query`, `/query/batch`).
- **Rules** — author, dry-run, and inspect rules; discover bindable fields
  (`/rules`, `/rules/:name/dryrun`, `/rules/catalog`).
- **Files** — upload and download blob-backed file fields (`/files`).
- **Preferences** — per-principal unit system, datetime pattern, timezone
  (`/prefs`).
- **Admin / control plane** — manage principals, capability grants, teams,
  tenants, and devices.
- **Extensions** — list, drive lifecycle, and inspect process/metrics/health
  (`/extensions`).
- **Jobs** — poll and cancel promoted background jobs and consume their progress
  channel (`/bulk/jobs`, `/ws/jobs/:id`).
- **Auth** — login and identity (`/auth/login`, `/auth/me`).

For the full, interactive endpoint reference — every path, schema, and
example, generated from the OpenAPI document — see the
[API Explorer](/reference/api-explorer). The raw spec is published at
[`/openapi.json`](/openapi.json).
