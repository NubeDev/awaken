## Goal
Stand up the standalone `rubix/` Cargo workspace and the SurrealDB store
boundary every other crate builds on. This is the root of the build: it
establishes the single store-and-brain engine (SCOPE principle 2, contract #6),
the project error model, the correlation-id type, and the durable read/write
handle through which all later persistence flows. Nothing else can land until
this is committed.

## Dependencies
None — this is the root workstream. WS-02..WS-04 consume the `rubix-store`
handle and the `rubix-core` error enum / correlation id introduced here.

## Scope / deliverables
- Crate(s): `rubix-core`, `rubix-store`, `rubix-server`.
- `rubix-core`: stable id type(s); the project error enum (`thiserror`) with
  `.context()`-style chaining; a `CorrelationId` type (minted at gate/ingest per
  contract #3); a `RuntimeConfig` struct (engine selection, namespace/database,
  profile).
- `rubix-store`: embedded SurrealDB boundary — `kv-mem` engine for tests and
  `kv-surrealkv` file-backed default selected by `RuntimeConfig`; namespace +
  database bootstrap; a schema-init seam (no schema yet); a health check; the
  durable read/write boundary handle; a scoped-session issuance **seam** (the
  real implementation lands in WS-03).
- `rubix-server`: minimal axum binary — `AppState` holding the store handle, a
  `GET /health` route, `main`.
- `scripts/check-file-size.sh`: the 400-line guard from FILE-LAYOUT.md §9
  (tracked `*.rs`/`*.ts`/`*.tsx`, excluding `src/generated/`).
- Workspace `Cargo.toml` with shared workspace deps: tokio, surrealdb, thiserror,
  serde, serde_json, axum, anyhow.
- Load-bearing contracts honored: #6 (SurrealDB is the one engine); seam for #1
  (scoped-session issuance) and #3 (correlation id) introduced.

### Assumptions / deviations
- SurrealDB 3.x typed verbs require `surrealdb::types::SurrealValue` (not serde)
  on stored types; the handle's `create`/`read` are bound on `SurrealValue` and
  the round-trip tests derive it.
- SurrealDB `select` on a never-created table errors rather than returning
  `None`; the isolation/missing-id tests materialise the table first, which is
  the real read pattern once tables exist.
- Env vars added (server only): `RUBIX_NAMESPACE`, `RUBIX_DATABASE`,
