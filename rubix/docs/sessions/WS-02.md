Status: Pending

## Goal
Deliver the schemaless document record and the `record→tagged→tag` graph that
replaces a fixed schema (SCOPE principle 4: generic, not domain-specific). The
platform processes data + tags, not a baked-in equipment/site/point ontology;
structure comes from tagging on the graph. This is the data substrate every
read, rule, and query above it operates over.

## Dependencies
WS-01 (committed). Consumes the `rubix-store` durable read/write handle for
SurrealQL execution and the `rubix-core` error enum / id type for record and
tag identity and error chaining.

## Scope / deliverables
- Crate(s): `rubix-core` (new `record` and `tag` modules).
- A generic `Record` type: id, namespace, free-form JSON content
  (`serde_json::Value`), created/updated timestamps; CRUD over SurrealQL through
  the `rubix-store` handle.
- A `Tag` type with CRUD.
- Create / remove the `tagged` graph edge (`record→tagged→tag`) via SurrealQL
  `RELATE` / edge delete.
- Tag-filter queries: find records matching a tag **set** (Haystack-style
  intersection) via SurrealQL graph traversal across multiple tags.
- Load-bearing contracts honored: #6 (graph + document in the one engine, no
  second store); SCOPE "generic, not domain-specific" — no fixed ontology.
