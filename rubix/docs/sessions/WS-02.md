Status: Pending
Verified: (re-verify on first action)

## Goal

Give suspended (awaiting-approval) agent runs an operator surface. STATUS.md AI-layer: HITL
escalation already returns a `SuspendTicket` and `POST /agent/chat` surfaces `status:
awaiting_approval` + `run_id`, but **the resume endpoint (operator approve/cancel) needs the
persistent-backend run store** — which does not exist yet. Inbound dispatch runs are likewise
"fire-and-log — no run registry/operator surface for in-flight dispatched runs."

This WS ships that store + the resume/cancel HTTP surface.

## Scope / deliverables

- `runs` table + store methods: persist a run on creation (chat or dispatch), update status
  (running/suspended/completed/cancelled), store the `SuspendTicket` for suspended runs.
- HTTP: `GET /api/v1/runs` (list, filter by status), `GET /api/v1/runs/{id}`,
  `POST /api/v1/runs/{id}/resume` (operator approve → re-drive the gated write),
  `POST /api/v1/runs/{id}/cancel`. Thin handlers → one domain fn each.
- Resume re-applies the suspended write through the priority array with gating re-checked.
- OpenAPI registration for the new routes + DTOs.
- Integration tests: suspend a run via the escalation band, list it, resume it (write lands), cancel
  another (store untouched).
