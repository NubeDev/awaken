# WS-03 — Parquet history `TableProvider` (object_store tiering)

Status: Pending
Verified: (re-verify on first action)

## Goal

History is SQLite-only today. STACK-DEISGN.md specifies a Parquet history store via `object_store`
(edge + cloud tiers) and a `his` DataFusion `TableProvider` over Parquet partitions. Ship the
Parquet write path + a `TableProvider` so `/query` and `/his/rollup` can read tiered history.

## Current state (re-grep before coding)

- `rubix/crates/rubix-query/src/sql/run.rs` + the query context registration — where canonical
  tables (`sites`, `equips`, `points`, `his`, `sparks`) register as `TableProvider`s over SQLite.
- `rubix/crates/rubix-query/src/rollup/run.rs` — time-bucketed rollups over `his`.
- `rubix/crates/rubix-server/src/store/his.rs` — current SQLite history writes.
- `rubix/crates/rubix-server/src/api/his/{insert,query,rollup}.rs` — HTTP his surface.
- DataFusion + `datafusion-table-providers` + `object_store` are the chosen deps (Apache-2.0) — add
  them; do not hand-roll a Parquet reader.

## Scope / deliverables

- A Parquet partition layout (by site/point/time bucket) written via `object_store` (local FS tier
  for edge, gated to keep the SQLite path as the live/recent tier).
- A `his` `TableProvider` that unions Parquet partitions (cold) with the SQLite recent tier (hot),
  so existing `/query` and `/his/rollup` keep working unchanged across the tier boundary.
- A rollover/flush path moving aged SQLite history rows into Parquet partitions.
- Env toggle for the Parquet tier (e.g. `RUBIX_HIS_PARQUET` + a path/store URL).
- Tests: write history → flush to Parquet → query reads across both tiers and rollup aggregates are
  correct over the union.
