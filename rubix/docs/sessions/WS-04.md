## Scope / deliverables

- Cargo features `edge` (default) and `cloud` on `rubix-server`; a runtime `Profile` enum selected by
  `RUBIX_PROFILE` (env) within what the compiled features allow (feature gates capability, runtime
  config selects among compiled options).
- A `Profile` config struct centralizing the per-profile defaults (store kind, history tier, whether
  drivers/supervisor launch, auth-required) — read once at boot, threaded into `AppState`.
- Edge profile preserves today's exact behavior (SQLite, drivers on, no auth) so nothing regresses.
- Cloud profile compiles and boots with the cloud defaults using only what exists today (Postgres
  and auth land in WS-05/WS-06 behind this seam — leave clearly-marked `// TODO(loop):` extension
  points ONLY where a later WS owns the impl, never a fake fallback).
- Tests: profile selection resolves correctly from env+features; edge defaults unchanged.

## Execution log

Task breakdown (single commit, one logical seam):
- `profile` module (`rubix-server/src/profile/`):
  - `kind.rs` — `ProfileKind` enum (`Edge`/`Cloud`), `FromStr`/`Display`, `UnknownProfile` error.
  - `config.rs` — `Profile` config struct + `StoreKind` (Sqlite/Postgres) and `HistoryTier`
    (LocalParquet/ObjectStore) enums; `Profile::defaults(kind)` centralizes per-profile defaults
    (store, history, `supervise_drivers`, `auth_required`).
  - `select.rs` — `resolve(Option<&str>)` / `select()`: feature-gated resolution. `compiled(kind)`
    is `cfg!(feature = ...)`; an unset `RUBIX_PROFILE` resolves to the compiled `default_kind()`
    (edge when its feature is on, else cloud); an uncompiled or unknown profile is a `ProfileError`.
  - `mod.rs` — barrel re-exports.
- `Cargo.toml` — `[features] default = ["edge"]`, `edge`, `cloud`.
- `lib.rs` — `compile_error!` when neither profile feature is set; `pub mod profile`; `AppState`
  gains a `profile: Profile` field.
- `main.rs` — `profile::select()` first at boot; `StoreKind::Postgres` fails closed with a clear
  "not available in this build" boot error (Postgres lands in WS-05 behind this seam); driver
  supervision gated on `profile.supervise_drivers`; `profile` threaded into `AppState`.
- Test harnesses + integration tests construct `AppState` with `Profile::defaults(ProfileKind::Edge)`
  so edge behavior is unchanged.
