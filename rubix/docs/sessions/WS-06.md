## Scope / deliverables

- JWT bearer middleware: validate signature against a configured JWKS/issuer (`RUBIX_OIDC_ISSUER`/
  `RUBIX_OIDC_JWKS`), extract subject + org/team/site claims, attach a `Principal` to request
  extensions. Fail-closed on the cloud profile, bypass on edge when auth is disabled.
- RBAC: org‚Üíteam‚Üísite hierarchy; an authorization check helper that gates each domain route by the
  principal's scope (reuse keyexpr/site scoping). Agent writes already gate by priority ‚Äî keep that;
  add the who-can-call-it layer above it.
- PATs / service accounts: a `tokens` table + issue/revoke; bearer path accepts a PAT as an
  alternative to an OIDC JWT.
- OpenAPI: mark routes' security scheme; tests for allow/deny across scopes + PAT path.
