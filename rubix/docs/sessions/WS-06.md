## Goal
Make user-facing definition mutations (dashboards, rules, tags, datasource config)
reversible — and only those: never the data plane (readings, insight firings),
never the audit log (SCOPE "Undo/redo"). Audit and undo derive from the same
captured change produced at the gate (WS-05): audit takes the immutable
projection, undo takes the mutable stack — one capture, two consumers
(contract #4). Undo applies the inverse back through the WS-05 gate, so it is
permission-checked and itself audited.

## Dependencies
- **WS-05** — the atomic before/after capture and the `Command::apply`
  gate pipeline. Undo consumes the captured change and re-enters the same gate.

## Scope / deliverables
- Crate(s): `rubix-gate` (undo module).
- A change record (forward + inverse) derived from WS-05's before/after capture,
  built only for definition mutations.
- A per-principal + per-resource undo stack — the mutable consumer of the capture,
  pushed on definition mutation and popped on undo; a redo stack fed by undo.
- `undo()` / `redo()` that re-apply the inverse/forward by constructing a WS-05
  `Command` and driving it through the gate — so the reversal is capability-checked
  and produces its own audit row.
- Boundary enforcement: undo refuses any data-plane record and the audit log,
  classified by table/record kind (definitions only).
- Load-bearing contracts honored (STACK-DEISGN.md): #4 (undo is the mutable stack
  derived from the same capture as audit; applied through the gate).

## Proposed file layout (verb-per-file, ≤400 lines)
```
crates/rubix-gate/src/
  undo/
    mod.rs                   # barrel
    change.rs                # ChangeRecord { forward, inverse } from capture
    classify.rs              # is-undoable? (definitions only; reject data/audit)
    push.rs                  # push onto per-principal+resource undo stack
    undo.rs                  # pop + apply inverse through the WS-05 gate
    redo.rs                  # pop redo + re-apply forward through the gate
    stack.rs                 # per-principal+resource stack (named concept, small)
crates/rubix-gate/tests/
  undo/
    change_test.rs
    classify_test.rs         # data-plane + audit refused
    undo_test.rs             # restores prior value, emits an audit row
    redo_test.rs
```
`stack.rs` is a named concept (the undo/redo stack), not a `types.rs`; the
boundary classifier lives in `classify.rs` as a verb, not a helper.
