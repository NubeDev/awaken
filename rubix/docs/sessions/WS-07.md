## Goal
Build the eventing spine's first two planes (SCOPE "Event bus"): in-process tokio
channels for component-to-component control events (no serialization, no network),
and SurrealDB live queries as the data-change / realtime pub-sub ("a record
appeared/changed"). Subscription scope is set once at subscribe, not per message:
live-query reads run on a WS-03 scoped session so SurrealDB row-level permissions
decide which records a principal sees (contract #1). Zenoh — the third plane — is
WS-12.

## Dependencies
- **WS-03** — the gate-issued scoped SurrealDB session the live-query subscription
  opens on (the once-at-subscribe scope decision).
- **WS-01** — `rubix-store` durable boundary the live query runs against.
- **WS-02** — the generic record/table the data-change plane observes.

## Scope / deliverables
- Crate(s): `rubix-bus`.
- An in-process broadcast surface over tokio channels for control events:
  publish + subscribe by event type, fan-out within the binary, no serialization.
- A live-query subscription surface over `rubix-store`: subscribe to a table on a
  WS-03 scoped session; record inserts/updates/deletes push events to the
  subscriber, filtered by SurrealDB row-level perms (scope set once, not per
  message).
- A principal sees all event *types* it is granted but only the *records* its
  perms allow — enforced by the data plane, not a per-message app proxy.
- Load-bearing contracts honored (STACK-DEISGN.md): #1 (reads/live run on the
  scoped session, filtered once at subscribe).
