## Goal
Deliver the unification and compute layer (SCOPE "DataFusion — query and
compute"). DataFusion sits ABOVE SurrealDB only where cross-datasource
unification or heavy vectorized aggregation is wanted (contract #6); SurrealQL
stays first. Provides the read-only SQL surface, vectorized time-window
aggregation that feeds rule decisions, and vector/semantic search over SurrealDB
vector columns.

## Dependencies
WS-01 (committed) for the `rubix-store` durable read/scoped-session handle; WS-02
(committed) for the `Record`/`Tag` model and canonical tables; WS-04 (committed)
for the capability-grant check — the query action is an app-enforced capability,
not a SurrealDB row-perm. Consumes the scoped SurrealDB session for reads.

## Scope / deliverables
- Crate(s): `rubix-query`.
- A SurrealDB DataFusion `TableProvider` exposing the canonical tables (records,
  tags, audit, insights) **read-only**, scanned through the scoped session.
- A unified query surface accepting read-only SQL (`SELECT`/`WITH` only) with an
  injection/statement guard rejecting any non-read statement.
- Vectorized time-window aggregation: `avg/min/max/sum/count/first/last` ×
  `minute…week` buckets, epoch-aligned, for feeding rule decisions.
- A vector / semantic-search surface over SurrealDB vector columns (k-nearest).
- The query action is gated by a WS-04 capability before execution.
- Load-bearing contracts honored (STACK-DEISGN.md): #2 (query is an app-enforced
  capability), #6 (DataFusion only for unification/aggregation, SurrealQL first).
