# WS-09 — Scoped zenoh session per driver + reference driver binary

## Goal

STATUS.md "Driver runtime": server-side queryables are owned-site scoped, but a *scoped zenoh
session per driver* (the driver's own session confined to its granted prefixes) was missing. Close
the gap and add the live out-of-grant refusal test STATUS.md "Known issues" flagged as absent.

## First action — current-state re-verification

Re-grepped every file:line the spec cited. Findings vs. the (partly stale) spec:

- `rubix-driver-sim` already exists and is the reference driver. It opened a **full** zenoh session
  and self-authorized each publish *manually* in `simulate.rs` (`caps.authorize_publish` per call) —
  no reusable scoped-session wrapper, and no subscribe-side gating, no test of an out-of-grant op.
- `rubix-driver` is deliberately IO-free (`lib.rs`: "No IO"); the zenoh-bearing wrapper cannot live
  there without breaking that contract. The genuine gap: a reusable session wrapper that enforces the
  `CapabilitySet` on every publish/subscribe, used by the sim, with a refusal test.

The spec's "Current state" claims held; no WS-doc drift correction needed beyond confirming the sim
was a full (not scoped) session.

## Task breakdown / commits

1. `ScopedSession` wrapper (`rubix-driver-sim/src/scoped.rs`): wraps a `zenoh::Session` + the
   driver's `CapabilitySet` + name; `put`/`declare_subscriber` authorize against the grant *before*
   touching the bus, returning `DriverError::Denied` on an out-of-scope key. Unit tests: publish and
   subscribe outside the grant are refused; in-grant publish is allowed.
2. Wire the sim through it: `main.rs` constructs the `ScopedSession`; `simulate.rs` publishes via the
   wrapper (removed the ad-hoc per-call manual check — the wrapper is now the single enforcement
   point). Behavior unchanged for the in-grant case.
3. Live end-to-end test (`tests/supervised.rs`): the supervisor spawns the sim configured to publish
   on a point *outside* its grant; the sim attaches (liveliness) but no `cur` ever reaches a peer
   subscriber — the scoped session refuses the publish locally. This closes the "no live
   out-of-grant refusal test" known issue.

## Assumptions / deviations

- The wrapper lives in `rubix-driver-sim`, not `rubix-driver`, because the contract crate is
  IO-free by design. The sim is the only driver binary today; a future shared `rubix-driver-runtime`
  crate can lift the wrapper when a second driver consumer appears (not built now — single consumer).
- `declare_subscriber`/`driver()` carry `#[cfg_attr(not(test), allow(dead_code))]`: the sim is
  publish-only so its binary never subscribes, but the wrapper enforces both directions per the spec
  and both are covered by the wrapper's tests.

## Follow-ups

- Lift `ScopedSession` into a shared driver-runtime crate when a second driver binary needs it.
- Bus-side per-driver session scoping (a server-issued session restricted to a driver's prefixes)
  remains a deeper change; today confinement is driver-side (local refusal) + server owned-site
  queryable scoping. No code path bypasses a guardrail.
