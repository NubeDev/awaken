## Goal
Deliver the Grafana datasource model (SCOPE "Datasources"): a datasource is a
declared, pluggable connection a query can read in a unified way, each
registering as a DataFusion `TableProvider`. "Unlimited datasources" means adding
a connector, not changing the core. Datasources are a kind of extension;
registering one is an app-enforced capability.

## Dependencies
WS-09 (committed) for the unified query surface and the native SurrealDB
`TableProvider` it spans; WS-04 (committed) for the datasource-register
capability. Consumes the DataFusion query entry from `rubix-query` and the
capability gate from `rubix-gate`.

## Scope / deliverables
- Crate(s): `rubix-datasource`.
- A `Connector` trait: declare config → produce a DataFusion `TableProvider`.
- A registry keyed by datasource id that the unified query surface reads from.
- Registration gated by a WS-04 capability (`datasource-register`).
- A Postgres connector via the `datafusion-table-providers` crate, behind a
  `postgres` cargo feature, failing closed when the feature/backend is absent.
- SurrealDB remains the native/default datasource (its provider lands in WS-09;
  this crate registers it as the default entry).
- Load-bearing contracts honored (STACK-DEISGN.md): #2 (register is
  app-enforced); SCOPE "Datasources" (pluggable connectors, each a
  `TableProvider`).

## Proposed file layout (verb-per-file, ≤400 lines)
```
crates/rubix-datasource/src/
  connector/
    mod.rs                 # barrel + Connector trait + DatasourceConfig
  registry/
    mod.rs                 # barrel + Registry (id -> provider)
    register.rs            # add a connector (WS-04 capability checked)
    authorize.rs           # datasource-register capability check
    resolve.rs             # look up a TableProvider by datasource id

    mod.rs                 # barrel
    connect.rs             # register the native SurrealDB datasource (default)
  postgres/                # #[cfg(feature = "postgres")]
    mod.rs                 # barrel
    connect.rs             # build a TableProvider via datafusion-table-providers

    register_test.rs

```
