# WS-10 — Datasource connector framework + Postgres connector

Status: Done
Started: 2026-06-14T22:29:24Z
Finished: 2026-06-15T02:30:00Z

## Goal
Deliver the Grafana datasource model (SCOPE "Datasources"): a datasource is a
declared, pluggable connection a query can read in a unified way, each
registering as a DataFusion `TableProvider`. Adding a connector, not changing the
core. Registration is an app-enforced capability (WS-04 `datasource-register`).

## Dependencies (verified committed)
- WS-09 (commit 93b8abef): unified query surface + native SurrealDB scan. The
  scoped scan is reused via the now-public `rubix_query::build_context`.
- WS-04 (commit 59d300d4): `Capability::DatasourceRegister` + `check_capability`.

## Task breakdown
1. Crate scaffold `rubix-datasource` + workspace member; `Connector` trait +
   `DatasourceConfig`.
2. Registry (id -> connector) + `register` (capability-gated) + `authorize`
   (datasource-register check) + `resolve`.
3. Native SurrealDB connector registered as the default entry; unified
   `query_registry` that builds the scoped SurrealDB context and registers each
   connector's provider on it, then runs read-only SQL spanning both.
4. Postgres connector behind `postgres` cargo feature via
   `datafusion-table-providers`; fails closed when the feature is absent.
5. Tests: unit (registry insert/lookup, authorize accept/reject) + integration
   (span SurrealDB + a second source, unregistered-id denied, ungranted register
   denied) + feature-gated Postgres round-trip skipped cleanly when `RUBIX_TEST_PG`
   unset.

## Minimal additive change outside the lane
- `rubix-query`: `build_context` widened from `pub(crate)` to `pub` and
  re-exported, so the datasource layer reuses the exact scoped scan rather than
  duplicating it. No behaviour change to WS-09 paths.

## Assumptions
- The unified surface "reads from the registry" = the registry produces the
  DataFusion `SessionContext` (SurrealDB canonical tables + connector providers)
  that a read-only query plans over.

## Env vars
- `RUBIX_TEST_PG` — Postgres URL enabling the feature-gated connector round-trip;
  unset skips it.
