## Goal
Deliver streaming ingestion (SCOPE "Ingestion and pre-processing"): sources
publish to Zenoh; pre-processing nodes consume in flight (decimate / filter /
enrich) and then persist to SurrealDB — raw high-rate streams are processed
before persistence, not written first and queried back. The permitted Zenoh
key-space is resolved by the gate ONCE at subscribe time (a WS-04 capability
decision), not per message, so high-throughput streams stay un-taxed. Data lands
append-only, partitioned by the edge identity (contract #5).

## Dependencies
WS-04 (committed) for the Zenoh key-space capability resolved at subscribe;
WS-02 (committed) for the `Record` model; WS-01 (committed) for the `rubix-store`
durable write handle and edge identity. Consumes the gate capability decision
once at subscribe and the store write boundary for persistence.

## Scope / deliverables
- Crate(s): `rubix-ingest`.
- A Zenoh subscriber whose key-space scope is resolved at subscribe via the gate
  (one capability decision, not per message).
- Pre-processing nodes applied in flight: decimate / filter / enrich.
- Persistence to SurrealDB as append-only records keyed by the edge-identity
  partition.
- Load-bearing contracts honored (STACK-DEISGN.md): #5 (append-only,
  edge-partitioned); #2 (Zenoh key-space is an app-enforced capability resolved
  once at subscribe).
