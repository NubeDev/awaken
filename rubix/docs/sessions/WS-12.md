## Goal
Deliver streaming ingestion (SCOPE "Ingestion and pre-processing"): sources
publish to Zenoh; pre-processing nodes consume in flight (decimate / filter /
enrich) and then persist to SurrealDB — raw high-rate streams are processed
before persistence, not written first and queried back. The permitted Zenoh
key-space is resolved by the gate ONCE at subscribe time (a WS-04 capability
decision), not per message. Data lands append-only, partitioned by the edge
identity (contract #5).

## What landed
New crate `rubix-ingest`, laid out one verb per file:

- `subscribe/authorize.rs` — `authorize_keyspace`: the single capability decision
  at subscribe. Checks the WS-04 `zenoh-subscribe` grant (fail closed), then
  confirms the requested key expression is *included* in the principal's
  edge-identity key-space subtree (`rubix/ingest/<namespace>/**`). Returns an
  `AuthorizedKeySpace` — the only way `listen` gets a scope, so a subscriber
  cannot open on an unauthorized key-space. Out-of-grant and out-of-partition
  both refuse here, once, before any Zenoh session opens.
- `subscribe/endpoint.rs` — `ZenohEndpoint`: the peer-session config (listen /
  connect endpoints + multicast scouting) → `zenoh::Config`.
- `subscribe/listen.rs` — `open_subscription` + `IngestSubscriber`: opens the
  Zenoh peer session, declares the subscriber on the authorized scope, decodes
  each sample's JSON payload. Holds the session alive alongside the subscriber.
- `subscribe/sample.rs` — `Sample`: the decoded shape (key + JSON content) the
  pre-processing nodes transform.
- `process/decimate.rs` — `Decimator`: stateful 1-in-N rate cut (factor clamped
  to ≥1 so it never swallows the whole stream).
- `process/filter.rs` — `Filter`: predicate drop over decoded content.
- `process/enrich.rs` — `Enricher`: merge derived/contextual fields into content
  (the one node that rewrites; never drops).
- `process/pipeline.rs` — `Pipeline`: composes decimate → filter → enrich in
  order, each stage optional, drop short-circuits the rest.
- `persist/partition.rs` — `partition_for` (the principal namespace = edge
  identity) + `keyspace_root` (the `<INGEST_ROOT>/<ns>/**` subtree) + `INGEST_ROOT`.
  One place the edge partition is derived, so subscribe scope and persistence
  partition cannot drift.
- `persist/append.rs` — `append_sample`: write a survivor as a fresh-id `Record`
  into the namespace partition (append-only; never updates an existing row; does
  not re-cross the command gate per sample — capability decided once at subscribe).
- `error.rs` — `IngestError` (Denied / KeySpace / Session / Sample / Persist),
  `From<IngestError> for rubix_core::Error`.

## Assumptions / deviations
- **Edge identity = principal namespace.** Contract #5 partitions on the edge
  identity; the platform's tenant/edge boundary is already the namespace
  (`Record::namespace`, `trace` rows). Ingest persists under the principal's
  namespace, so two edges (distinct namespaces) never collide. No new identity
  concept was introduced.
- **Key-space scope = `rubix/ingest/<namespace>/**`.** WS-04's `Grant` binds a
  capability to a (subject, namespace), with no per-key-space field. The grant
  confers the right to subscribe; the *scope* is bounded to the principal's
  edge-identity subtree at authorize time. This keeps the capability decision a
  single check (the grant) plus a structural inclusion test, both fail closed,
  with no new grant schema. A finer-grained per-key-space grant model, if ever
  needed, is a WS-04 change, not an ingest hack.
- **Ingest persists directly via `create_record`, not through the command gate.**
  The capability is decided once at subscribe (contract #2: high-throughput
  streams stay un-taxed); routing every high-rate sample back through the
  per-command gate would defeat the streaming design. Data-plane ingest is the
  gate's "minted at ingest" correlation chokepoint, distinct from the command
  path. Append-only writes carry no before-image to capture.
- **Zenoh features.** `zenoh = { default-features = false, features =
  ["transport_tcp"] }` — TCP is the deterministic edge/loopback transport;
  the heavier default transports (QUIC/TLS/WS/UDP/serial) are left out of the
  edge build and can be feature-gated on later if a deployment needs them.
- **Test runtime.** The listen test uses `#[tokio::test(flavor =
  "multi_thread")]` because the Zenoh runtime rejects the current-thread
  scheduler.
