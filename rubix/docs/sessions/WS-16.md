## Goal
Wire every crate into the running binary and expose it (SCOPE "Frontend",
"Preferences"). Every *mutation* is routed through the WS-05 gate; every *read*
runs on a WS-03 scoped session; realtime is the WS-07 live-query feed bridged to
WebSocket clients on their scoped session (contract #1). OpenAPI 3.1 is served via
utoipa, and `rubix-prefs` applies per-user units + datetime formatting at the
response DTO layer.

## Dependencies
- **All prior WS** — this is the integration layer. Specifically: WS-05 (gate) for
  mutations, WS-03 (scoped session) for reads, WS-07 (live queries) for the WS
  bridge, WS-08/09 (rules/query), WS-10 (datasources), WS-13 (extensions/control),
  WS-14 (profile selection into `AppState`).

## Scope / deliverables
- Crate(s): `rubix-server` (transport) + `rubix-prefs`.
- axum HTTP routes, **one file per route, handler ≤20 lines** (extract → call
  domain → map DTO → return), for records / tags / query / rules / insights /
  datasources / extensions — mutations dispatched through the WS-05 gate, reads run
  on the WS-03 scoped session.
- A JSON-RPC control endpoint for extension/control actions (WS-13).
- A WebSocket bridge streaming WS-07 live-query events to a client on its scoped
  session (scope set once at subscribe).
- OpenAPI 3.1 via utoipa served at `/api-docs/openapi.json`.
- `rubix-prefs`: per-user units (metric/imperial) + datetime formatting applied at
  the response DTO layer.
- Load-bearing contracts honored (STACK-DEISGN.md): #1 (commands → gate, reads →
  scoped session); all cross-cutting contracts surface at this integration layer.
