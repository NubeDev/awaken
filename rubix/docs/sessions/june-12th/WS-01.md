## What landed

The node gained an awaited mode alongside the existing fire-and-forget default, selected by an
`await: true` board-node config flag. Detached stays the control-board default (a control board must
not block on an LLM); awaited blocks the single-shot run on the agent and surfaces its decision so a
downstream node can branch on it.

- `rubix-flow` `port.rs`: new `AgentOutcome { run_id, response, steps }` (decoupled from the awaken
  runtime types, same boundary as `PointAccess`) and a `request_agent_blocking` trait method
  (default fails closed, like `request_agent`). Exported from `lib.rs`.
- `rubix-flow` `node/agent_call.rs`: `await` config flag routes to `request_agent_blocking` and
  emits the agent's `response` text on `output`; the detached path is unchanged.
- `rubix-flow` `board/run.rs`: the single-shot runner now drains outputs until a poll interval is
  quiet (settled) or a bounded `MAX_SETTLE` deadline, so an awaited node that emits after its LLM
  round-trip is still collected. A plain control board still settles in one interval.
- `rubix-server` `flow/access.rs`: `StorePointAccess::request_agent_blocking` bridges the sync port
  to the async runtime via `block_in_place` + `Handle::block_on` (multi-thread flavor required);
  shared `activation_for`/`agent_or_fail` helpers dedupe the detached and awaited paths.

## Tests

- `rubix-flow/tests/board.rs`: awaited `agent_call → write_point` surfaces the canned decision at the
  terminal write node (an intermediate node's outport is consumed by the connector, so the decision
  is observed where it lands); awaited call without an agent fails closed on `error`.
- `rubix-server/tests/api_tests/agent_call.rs`: a stored awaited board run drives a scripted agent
  whose `write_point` tool commands a point through the priority array *within* the board run — no
  post-run polling — and the node surfaces an `output` packet.

## Assumptions / deviations / follow-ups

- An intermediate node's `output` packet is consumed by the reflow connector that forwards it, so it
  is not separately observable in the run outputs; the decision is asserted at the terminal node it
  reaches. This is existing reflow behavior, not introduced here.
- The agent's gating (`RUBIX_AI_MIN_PRIORITY` / escalation band) is enforced inside the agent's own
  tools, unchanged; an awaited `agent_call` inherits it for free.
