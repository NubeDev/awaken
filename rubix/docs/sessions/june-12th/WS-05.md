## Goal

STACK-DEISGN.md: "Postgres (cloud), SQLite (edge)". Today the store is SQLite-only. Add a Postgres
store implementation behind the WS-04 `cloud` profile, selected at runtime, with SQLite remaining the
edge default. Cloud relational tables (users/teams/config) per STATUS.md "Postgres federation".

## Scope / deliverables

- A store abstraction (trait) covering the methods the server uses; SQLite impl (existing, refactored
  behind it) + a Postgres impl (sqlx or tokio-postgres + a pool) gated by the `cloud` feature.
- Postgres schema/migrations mirroring the SQLite schema (sites/equips/points/his/sparks/boards/
  widgets + WS-02's runs). Respect the ADR persistence guardrails (A-G2 RunRecord validation, etc.).

## Outcome (commit 200269f6)

Rather than an async-trait rewrite (which would force `.await` into ~70 synchronous call sites and
destabilize the shared branch), the store keeps its synchronous r2d2-pooled shape and gains a second
backend behind a `Backend` enum (`Sqlite(pool)` / `Postgres(pool)`, the latter cfg `cloud`). Each
public `Store` method matches the backend: the SQLite body stays inline, the Postgres body lives in a
parallel `store/postgres/<resource>.rs` module. Backend-agnostic logic (priority-array command/ingest
semantics, keyexpr prefix split, widget kind tokens, suspended-run guard) was extracted into shared
`pub(crate)` fns so both backends reuse one implementation.

### Task breakdown
- `Cargo.toml` workspace + `rubix-server` deps: `postgres = "0.19"`, `r2d2_postgres = "0.18"`,
  `cloud = ["dep:postgres", "dep:r2d2_postgres"]`.
- `store/backend.rs` (new): `Backend` enum + `SqliteConn`/`PostgresConn` pooled-conn aliases.
- `store/open.rs` (rewrite): `Store { backend }`; `open` (SQLite/edge), `connect` (dispatch on
  `postgres://`/`postgresql://` prefix), `is_postgres_target`, `connect_postgres` (cloud: parse
  `postgres::Config`, build pool, run `SCHEMA_POSTGRES`; non-cloud: bail "requires --features cloud"),
  `sqlite_conn`/`postgres_conn`, `truncate_all_for_tests` (cloud, test-only).
- `store/schema.rs`: `SCHEMA` → `SCHEMA_SQLITE`; added cloud `SCHEMA_POSTGRES` (TEXT ids/timestamps,
  BIGINT version/steps, BOOLEAN flags; same tables/constraints/indexes).
- `store/error.rs`: cloud `From<postgres::Error>` mapping UNIQUE/FK violations → `Conflict`, else `Db`.
- 11 resource files dispatched; `store/postgres/{mod,codec,sites,equips,points,command,his,his_flush,
  keyexpr,boards,widgets,sparks,runs}.rs` (new) hold the `$N`-placeholder Postgres bodies and shared
  row decoders. Ids bound as `to_string()` / parsed with `Uuid::parse_str` (sidesteps postgres'
  uuid-0.8 vs workspace uuid-1.x conflict).
- `store/main.rs`: per-profile `default_db` (`postgres://localhost/rubix` under cloud), `Store::connect`,
  warn when cloud expects Postgres but the target isn't; query engine disabled for Postgres targets.
- `tests/store_suite.rs` (new): one `run_suite(store)` contract (sites/equips/points/command/his/sparks/
  widgets/boards/runs incl. conflict, FK, keyexpr both directions, priority write+relinquish, batch his,
  spark filter+ack, board versioning, run suspend/settle one-shot, cascade delete). SQLite always;
  cloud-feature Postgres pass skips cleanly (eprintln, no `#[ignore]`) when `RUBIX_TEST_PG` unset.

### Validation
Edge build + clippy clean, cloud (`--no-default-features --features cloud`) build + clippy clean, full
workspace clippy clean, `cargo test --workspace` green (126 SQLite-path tests on edge). The shared

### Assumptions / deviations
- Synchronous Postgres (`postgres` crate) chosen over async sqlx/tokio-postgres to avoid churning the
  server's synchronous call surface on a shared branch.
- UUID/timestamp columns are TEXT in both backends for identical round-tripping, not native PG types.
