## Design notes / decisions

- **JWT path** uses the already-present `jsonwebtoken` crate (`DecodingKey::from_jwk`),
  JWKS fetched once at boot via `reqwest` (rustls). Symmetric (HMAC) algorithms are
  rejected ‚Äî a JWKS public key signed with HMAC would be a forgery vector.
- **PATs** are `rbx_pat_<id>.<secret>`; only the sha-256 of the secret is stored
  (`UNIQUE(secret_hash)` ‚Üí single indexed verify lookup). 122-bit secret from
  `uuid` v4 (OS CSPRNG) ‚Äî no new RNG dependency.
- **RBAC** `Scope.covers` mirrors `Capability::covers` (a broader grant covers a
  narrower request, never a sibling). Domain resources are gated by `covers_resource`
  (org + site), because a domain `Site` has org+slug but no team ‚Äî a team is a
  *principal* grouping, and the team‚Üísite ownership table is tenancy data WS-07
  owns. Token issuance/revocation gate on the full org/team/site `covers`.

## Follow-ups (not blockers)

- Per-route scope gating is applied at the **site** routes (the scope root) and
  the **token** admin routes. Equip/point/board/spark/his/query/widget routes
  currently inherit only the authenticated-caller requirement from the
  middleware; tightening them needs each handler to resolve its owning site
  first (mechanical, deferred to keep this WS focused on the auth/RBAC core).
- Live OIDC-JWT signature verification is unit-tested for the fail-closed paths
  (garbage token, empty JWKS, HMAC rejection); a full happy-path JWT test would
  need an RSA keypair fixture. The PAT path is end-to-end integration-tested,
  exercising the same middleware + gate + store. A live-issuer JWT test is a
  later addition (parallels the "no live LLM provider test" gap for the agent).
- A team‚Üísite ownership model (WS-07 tenancy) would let `covers_resource` honor
  team scoping; today team narrows token admin but not resource reads.
