## Goal

Give a dashboard variables: a variable bar lets a user pick `$site`/`$equip`/`$datasource`/… ;
widgets re-query against the selection; values deep-link via `?var-*`. One dashboard then serves a
whole fleet instead of one board per site. Variables live in the dashboard's stored JSON (travel
with export/import — no separate table).

## Scope / deliverables

- **Model:** `variables: Vec<Variable>` on `Dashboard`; the `VariableKind` closed enum — `constant`,
  `custom`, `query`, `datasource`, `site` (headline, resolves to `site_id`), `interval`, `textbox` +
  read-only built-ins (`$__org`/`$__site`/`$__user`; `$__from`/`$__to` arrive with WS-03). DTO derives
  `ToSchema`; register in `api/openapi.rs`. TS mirror in `types.ts`.
- **Resolution** (`ui/src/features/variables/resolve.ts`, new): order built-ins → constants/custom →
  site/datasource → query (topological); cascading (a `query` var's SQL references another var);
  **detect + reject cycles** with a clear error; cache option lists per `(name, parent-values)`.
- **Re-query:** introduce `varRevision` (hash of resolved values), fold into the widget *data* query
  key for any var-referencing widget. Unreferenced widgets unaffected (back-compat).
- **Variable bar** (`ui/src/features/variables/**`, new): single/multi-select + textbox, "All" when
  `include_all`. **Editor:** add/edit/reorder/delete, choose kind, author the option query, preview
  resolved options, set multi/include-all/hidden.
- **URL state:** `?var-site=A&var-site=B` (repeatable for multi); restore on load; shareable. Reserve
  the `var-` prefix for explicit variable state (bare params belong to WS-06's context source).
- **Repeat-by-variable hook:** expose the resolved value list (do not render repeat here).
