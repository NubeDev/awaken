## Goal

The user-facing time control: a dashboard-level time-range picker + refresh control whose
`{from, to, interval}` flows into every widget query, with shareable URL state, auto-refresh, and
zoom-by-drag. This converts the dashboard from a static grid into a live, interactive surface.

## Scope / deliverables

- **`ui/src/stores/time-store.ts`** (new): `{ from, to }` each an absolute ISO instant **or** a
  relative token (`now`, `now-6h`, `now/d`), plus `{ refresh }` (off | 5s | 10s | 30s | 1m | 5m | 15m).
- **Relative-range resolver:** turns relative tokens into concrete `{fromTs, toTs}` at query time,
  incl. `now/d` rounding. (Server resolves `now` authoritatively per WS-03; client resolver mirrors.)
- **`ui/src/features/time/**`** (new) TimeRangePicker: quick ranges (5m/15m/1h/6h/24h/7d/30d, Today,
  Yesterday), absolute from/to + calendar, relative input, refresh-interval dropdown, manual refresh.
- **Auto-refresh loop:** invalidate widget data queries on the chosen interval (bump a `tick` floor
  folded into the key, or TanStack `refetchInterval`); **pause when the tab is hidden**.
- **Wire into queries:** pass `time_range` + `interval_secs` (WS-03 DTOs) to the widget data fetch;
  for `point_history` widgets thread the resolved range into the `his` `start`/`end` params.
- **Cache-key snapping:** fold resolved `{fromTs, toTs}` **snapped to the refresh tick** (not raw ms —
  raw busts cache every render) + `interval_secs` into the widget data query key.
- **URL state:** `?from=&to=&refresh=` round-trips, shareable (reuse WS-02's URL mechanism).
- **Drag-zoom:** dragging a range on a history/line chart sets the global range, with zoom-out/back.
- **Per-widget override (stretch, low priority):** leave the model room for `timeShift`/relative
  override; do not block the WS on it.
