## Goal

Build the one append-only **change ledger** substrate that powers audit log + undo/redo "for
everything": the `changes` table, the per-actor `undo_cursors` table, the sole-write-path
`ChangeRecorder`, the `Reversible` registry with a generic snapshot reverser, and the coverage guard
that fails on a silently-unwired kind. This WS lands the substrate **without** wiring every handler —
that is WS-08.

## Scope / deliverables

- **`changes` table** (append-only, one row per logical mutation): `id, at, org, site_id?, actor,
  kind, resource_id, op (Create|Update|Delete), before? (full JSON snapshot), after?, group_id,
  correlation?`. `Actor = User{subject} | Agent{run_id, model} | System`. Indexed on `(org, at)`,
  `(org, kind, resource_id)`, `group_id`. **Snapshot, not patch.**
- **`undo_cursors` table:** `(org, subject)` → cursor position, undo/redo is **per-actor**; a
  version/epoch column for CAS-guarded concurrent undo.
- **`ChangeRecorder`** (`store/changes.rs`, new) — the **sole** write path; `record(Change)` runs
  **inside the mutation's DB transaction** so the change commits atomically with the mutation.
- **`Reversible` registry:** `trait Reversible { kind(); apply_inverse(store, &Change); apply_forward
  (store, &Change); }`. A **generic snapshot reverser** covers most kinds mechanically (undo Create→
  delete, undo Delete→re-insert `before`, undo Update→write `before`; redo = forward). Custom impls
  only where a cascade/side-effect needs it.
- **Cascade grouping support:** the substrate must let a handler record **one row per deleted child
  under a shared `group_id`** so a subtree undoes as one step (WS-08 wires the site path; this WS
  provides the `group_id` API + the grouped-undo dispatch).
- **Coverage guard test:** enumerate registered reversible kinds; assert each has ≥1
  create/update/delete path producing a `changes` row, and `before` is non-null on update (catches
  recording outside the tx). A registered mutable kind with no recording path **fails the test**.

## Notes / boundaries
- **Wiring `record()` into each mutation handler, secret redaction, the agent actor, the audit-read
  routes, and the `/undo`/`/redo` HTTP surface are all WS-08.** This WS ships the substrate + the
  registry + the guard only.
- Point *value* writes (`cur`/`write`) are telemetry → `his`, **not** the audit ledger. Out of scope.
