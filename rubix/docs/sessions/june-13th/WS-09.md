## Goal

The audit/undo **frontend**: undo/redo control + shortcuts that refresh exactly the touched queries,
a toast with inline undo after a mutation, a per-resource History tab (timeline + before→after diff),
and an admin Audit log screen gated behind the audit capability.

## Current state (re-grep before coding)

- `ui/src/components/layout/page-header.tsx` — currently bare; mount the undo/redo toolbar button here.
- `ui/src/api/keys.ts` — TanStack query keys (e.g. `qk.widgets(...)`); the `/undo` response's touched-
  resource ids map to these for precise invalidation.
- `ui/src/api/types.ts` — `Change`/`Actor`/`Op` mirrors (WS-08).
- `ui/src/features/builder/` — the dashboard builder; the History tab hangs off resource detail views.

## Scope / deliverables

- **Undo/Redo control + shortcuts:** Cmd/Ctrl+Z and Shift+Cmd/Ctrl+Z + a toolbar button in
  `page-header.tsx`. On success, use the returned touched-resource ids to invalidate the matching
  TanStack keys (e.g. `qk.widgets(...)`) so the canvas refreshes without a reload.
- **Toast with inline undo** after a mutation ("Deleted dashboard · Undo"), carrying the `group_id`.
- **Per-resource History tab:** a timeline + before→after diff for one resource (powers a "History"
  tab on a dashboard, datasource, rule, …), reading `GET /api/v1/audit/{kind}/{id}`.
- **Admin Audit log screen:** the filters from `GET /api/v1/audit` (kind/resource_id/actor/op/time
  range), paged newest-first, before→after diff rendering — **gated behind the audit capability** (the
  screen/route is hidden/denied for a non-admin, matching the server's capability gate).
