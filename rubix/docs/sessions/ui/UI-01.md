## Goal

Every type in `ui/src/api/types.ts` was written by reading Rust source, not by testing the wire.
Several are guesses. Verify the whole client surface against the running server's
`/api-docs/openapi.json` and real responses, and fix every mismatch. Later workstreams depend on
this being the truth — a wrong shape here surfaces as a silently-blank panel after UI-03.

## Verified wire truth (re-grepped Rust source — utoipa derives schema from these exact types)

- `PointValue` (`rubix-core/src/value.rs:9` `#[serde(untagged)]`) → `boolean | number | string`. types.ts already correct.
- `PriorityArray` (`rubix-core/src/priority.rs:16`) → `{ slots: (PointValue|null)[], relinquish_default: PointValue|null }`. types.ts already correct.
- `TagSet` (`rubix-core/src/tags.rs:12` `#[serde(transparent)] BTreeMap<String, serde_json::Value>`) → a JSON **map**, NOT `string[]`. types.ts WRONG (`tags: string[]`). Fixed: `TagSet = Record<string, unknown>`, plus `api/tags.ts` (`tagNames`, `hasTag`) for the marker-presence reads components do.
- `RunRecord` (`rubix-server/src/agent/run_record.rs:117`) → real run row: `{ id, thread_id, origin: RunOrigin, status: RunStatus, response, steps, pending_write?: PendingWrite, created_at, updated_at }`. `RunSummary` was invented. Renamed to `RunRecord` + added `RunOrigin`/`RunStatus`/`PendingWrite`.
- `QueryResponse` (`rubix-server/src/api/query/run.rs:20`) → `{ rows: Value[] }` (array of JSON row-objects), NO `columns`. types.ts WRONG. Fixed to `{ rows: QueryRow[] }` where `QueryRow = Record<string, PointValue | null>`. (history/index.tsx already reads `rows` as objects and derives columns — reconciled.)
- `WriteSource` (`rubix-server/src/api/command/source.rs:11` snake_case) → `operator | agent` only. types.ts WRONG (`schedule` invented). Removed `schedule`.
- `WriteRequest` (`rubix-server/src/api/command/write.rs:17`) → `value` required, `priority`/`source` serde-defaulted (UI may omit). Kept optional.
- `ChatResponse` / `ChatStatus` (`rubix-server/src/api/agent/chat.rs`) → already correct (`completed | awaiting_approval`).
- `ErrorBody` (`rubix-server/src/error.rs:44`) → `{ error: string }`. client `readError` already reads `.error`.
