# UI-01 — Wire-contract truth-up: TS types verified against OpenAPI

Status: Pending

## Goal

Every type in `ui/src/api/types.ts` was written by reading Rust source, not by testing the wire.
Several are guesses. Verify the whole client surface against the running server's
`/api-docs/openapi.json` and real responses, and fix every mismatch. Later workstreams depend on
this being the truth — a wrong shape here surfaces as a silently-blank panel after UI-03.

## Current state (verify first — may have drifted)

- `ui/src/api/types.ts` — hand-written mirrors. Known-suspect:
  - `PriorityArray` — assumed `{ slots: PrioritySlot[], relinquish_default }`. The Rust type is
    `rubix-core/src/priority.rs`; check its actual serde output (field names, slot encoding,
    whether `PointValue` is untagged).
  - `RunSummary` — invented (`id/status/title/started_at`). The real shape comes from
    `rubix-server/src/api/runs/` DTOs.
  - `QueryResult` — assumed `{ columns, rows }`. Check `api/query/` response.
  - `WriteSource` — assumed lowercase strings; check serde casing on the Rust enum.
  - `Spark.point_ids`, `TagSet` serialization (`tags: string[]` — is it an array or a map?).
- `ui/src/api/client.ts` — error envelope assumed `{ error }` or `{ message }`; check the real
  `ErrorBody` in `rubix-server/src/error.rs`.
- `ui/src/api/endpoints.ts` — query-param names (`site_id`, `equip_id`, `tags`) vs `ListPointsQuery`.

## Tasks

1. Run the server, fetch `/api-docs/openapi.json`, and diff every schema the UI consumes against
   `types.ts`. Exercise each endpoint with `curl` against a scratch SQLite store (create a site,
   equip, point; write; relinquish; spark; query) and compare the actual JSON to the types.
2. Fix `types.ts` (and any component reading a changed field). One commit per corrected shape if
   the fix ripples into components.
3. Make the error path real: `ApiError` must surface the server's `ErrorBody` message for 4xx,
   and hooks must propagate it (sonner toast on mutation failure is already wired in `main.tsx`).
4. Extend `client.unit.test.ts` / add `types`-level tests asserting the corrected decode of one
   captured real-response JSON per resource (commit captured JSON under `src/api/__fixtures__/`
   — test-only data, allowed).

## Out of scope

Codegen (openapi-typescript) — allowed if it produces a *smaller* diff than hand-fixing, but the
generated file must land under `src/api/generated/` and `types.ts` must re-export the names the
app already imports. Don't rename app-wide.

## Done means

- Every UI-consumed schema demonstrably matches the wire (captured-response tests pass).
- `pnpm build` + `test:unit` green. No component renders `undefined` where the demo showed data.
