## Goal

The wiresheet currently renders a static TS graph (`features/flows/data/sample-board.ts`) under
chrome that asserts "deployed В· running В· v4" unconditionally. Make Flow Boards render **stored
boards from the API** and delete the static graph.

## Wire facts (Verified 2026-06-13 by UI-04)

- `GET /api/v1/boards` в†’ `BoardView[]`: `{ id, slug, version, display_name, enabled, trigger,
  graph, created_at }` (`rubix-server/src/api/boards/dto.rs:33`). `created_at` is an RFC3339
  string; `trigger` is `{ kind: 'manual' } | { kind: 'interval', seconds } | { kind:
  'subscription', key }` (`scheduler/trigger.rs`, serde tag `kind`, snake_case).
- `graph` is a `rubix_flow::BoardGraph` = `{ nodes: BoardNode[], connections: BoardConnection[] }`
  (`rubix-flow/src/board/schema.rs:11`). `BoardNode = { id, component, config: object }`.
  `BoardConnection = { from_node, from_port, to_node, to_port }`. There is **no `ports` array,
  no `wires` field, and no canvas geometry** (node x/y does not exist on the wire). The original
  spec's "nodes/ports/wires" was wrong; corrected here.
- Valid `component` names: `read_point`, `write_point`, `query_his`, `emit_spark`, `agent_call`,
  `trigger` (`rubix-flow/src/board/registry.rs:15`).
- `POST /api/v1/boards/{slug}/run` runs the stored board в†’ `RunBoardResponse { outputs:
  NodeOutput[] }` where `NodeOutput = { node, port, value }` (`board/run.rs:30`). 404 if no such
  slug (`run_stored.rs`).
- UI-02's seed creates sites/equips/points/sparks only вЂ” **no board**. UI-04 extends the seed to
  insert one Manual board (backend gate applies).
