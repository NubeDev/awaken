# UI-04 — Flows on stored boards (`/api/v1/boards`)

Status: Pending
Depends: UI-03

## Goal

The wiresheet currently renders a static TS graph (`features/flows/data/sample-board.ts`) under
chrome that asserts "deployed · running · v4" unconditionally. Make Flow Boards render **stored
boards from the API** and delete the static graph.

## Wire facts (verified 2026-06-13 — re-verify)

- `GET /api/v1/boards` → `BoardView[]`: `{ id, slug, version, display_name, enabled, trigger,
  graph, created_at }` (`rubix-server/src/api/boards/dto.rs`).
- `graph` is a `rubix_flow::BoardGraph` (nodes/ports/wires; schema in `rubix-flow/src/board/`).
  **It has no canvas geometry** — node x/y does not exist on the wire today.
- `POST /api/v1/boards/{slug}/run` runs the stored board (`RunBoardResponse`).

## Tasks

1. Client: add `boards.list / get / runStored` endpoints + types matching `BoardView` and the
   actual `BoardGraph` JSON (inspect a seeded board — UI-02 should seed at least one board; if it
   didn't, extend the seed in this WS, backend gate applies).
2. Board picker: left of the canvas (above the palette or a select in the status bar) listing
   stored boards by `display_name`; selection loads its graph.
3. Graph mapping: `BoardGraph` → React Flow nodes/edges. Geometry: deterministic auto-layout
   (layered by wire topology — sources left, sinks right, like the current sample looks). Keep
   `FlowNode` visuals exactly as they are (the look is frozen). Node kind → icon/kind mapping
   from the node type names in `rubix-flow/src/node/`.
4. Status bar honesty: `enabled` → "enabled/disabled" badge (green/muted), real `version`,
   real node/wire counts. "deployed · running" text goes away unless a real run-state source
   exists. **Test run** button → `POST /boards/{slug}/run`, surface `RunBoardResponse` (toast +
   inspector pane). **Deploy/Versions** buttons: keep visually but disabled with tooltip
   "publishing lands with the editor" (an honest disabled control, not a fake success).
5. Delete `data/sample-board.ts`; grep gate now covers it.
6. Editing/saving graphs (drag-from-palette → `POST /boards`) is **out of scope** — palette stays
   as the visual catalogue it is. Log the editor as a follow-up in this doc, not TODOs.md.

## Done means

Flows page lists real stored boards, renders the seeded board's real graph with the frozen node
look, Test run executes it against the real engine, and no static graph file exists.
