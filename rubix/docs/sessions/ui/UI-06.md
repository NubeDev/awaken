- `rubix-server/src/api/widgets/{mod,create,list}.rs` ‚Äî only `POST` (201 `Widget`) and
  `GET` (`[Widget]`, `?site_id=`) on `/api/v1/widgets`. **No DELETE route.**
- `rubix-server/src/store/widgets.rs` ‚Äî `create_widget` + `list_widgets` only; no

- `rubix-core/src/model.rs` ‚Äî `WidgetKind = point_value | point_history | board_output`;
  `Widget { id, site_id, kind, title, target, created_at }` ‚Äî **no layout/size/order field**.
- Point keyexpr the server stores is `org/slug/equip.path/point.slug` (no `/cur`), per
  `__fixtures__/point.json` (`acme/hq/ahu-3/sat`). `point-detail.tsx` appends `/cur` for display
  only; widget targets use the bare server form so they round-trip.

## Assumptions / deviations

- **No removal verb shipped.** The canvas has no Remove (X): the wire exposes no
  `DELETE /api/v1/widgets/{id}`, and a localStorage "hidden" flag would hide a real server row
  (a fake). Logged to `../TODOs.md`; canvas is create / render-live / pin only. No dead button.
- **Layout is order-by-creation.** No layout/size field on the wire; drag-reorder deferred rather
  than shipping a half-persistent localStorage-only arrangement. Logged to `../TODOs.md` proposing
  a server-side layout field.
- `board_output` tiles run the board on each poll (no read-only output endpoint). This executes the
  board's actors; acceptable for the MVP since stored boards in the seed are pure compute, and it is
  the only real output source. Noted here as a follow-up candidate.
