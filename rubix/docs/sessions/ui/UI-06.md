# UI-06 — Dashboard Builder MVP on `/api/v1/widgets`

Status: Pending
Depends: UI-03

## Goal

Replace the Builder stub page with a working compose surface persisted through the real widgets
API. MVP scope: pin, arrange, persist, render live — not the full mockup editor.

## Wire facts (verify first)

- `GET/POST /api/v1/widgets` (+ delete?) — read `rubix-server/src/api/widgets/` for the actual
  verbs and the `WidgetKind` enum: `point_value | point_history | board_output`, `target` is a
  point keyexpr or board slug (`rubix-core` `WidgetKind` docs). If the record has no
  layout/size fields, layout state is **client-side per browser** (localStorage) for the MVP and
  a `../TODOs.md` entry proposes persisting layout server-side. Do not invent wire fields.

## Tasks

1. Client + hooks for widgets CRUD (UI-01 typing discipline: verify shapes on the wire).
2. **Palette rail** (left, mockup look): the widget kinds that map to real `WidgetKind`s —
   "Live Value" (point_value), "Line / Area" (point_history), "Board Output" (board_output).
   Mockup-only kinds with no wire backing (Gauge, Floor Map, …) appear disabled with a tooltip,
   or not at all — pick one, consistently. Nothing fake-clickable.
3. **Add flow**: choosing a kind opens a binder dialog — site→equip→point cascade (reuse the
   real points hooks) or board slug for board_output → `POST /widgets`.
4. **Canvas**: CSS grid of widget cards rendering **live data**: point_value → current value +
   freshness (reuse the KPI/stat card visuals); point_history → the existing history area chart;
   board_output → latest run output (`boards.runStored` result or stored output endpoint if one
   exists). 5s polling like the rest of the app. Remove (X) → DELETE (or the real removal verb).
5. **Pin from Points**: "Pin" action on `point-detail.tsx` creating a widget for the open point —
   the loop the mockup promises (point → pin → dashboard builder).
6. Keep the frozen look: card chrome, eyebrow labels, density per the reference shots. Drag
   re-order is nice-to-have (dnd via CSS grid order in localStorage); skip if it threatens the
   timeline — order-by-creation is acceptable for MVP.

## Done means

Create, render-live, and remove widgets against the real API; pins survive reload (server rows);
Builder page matches the house look with zero invented data.
