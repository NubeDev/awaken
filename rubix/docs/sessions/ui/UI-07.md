# UI-07 — Cleanup & hardening: template leftovers, auth header, tests, lint

Status: Pending
Depends: UI-04, UI-05, UI-06 (runs last; it deletes and locks)

## Goal

Ship-shape: remove the shadcn-admin template residue the earlier rows didn't own, wire the
minimal real-auth hook, and make the gates permanent.

## Tasks

1. **Template residue audit** (delete or make real — nothing in between):
   - `features/auth/` + `routes/(auth)/` (sign-in/sign-up/otp/forgot-password) and
     `stores/auth-store.ts`: the backend has OIDC/JWT + tokens (backend WS-06), but the UI has
     no configured issuer yet. Decision pre-made: **keep `sign-in` only**, stripped to a single
     "API token" field that stores a bearer token; delete sign-up/otp/forgot-password/sign-in-2
     and their routes/tests. Full OIDC redirect flow = follow-up noted here, blocked on
     deployment config (`../TODOs.md`).
   - `components/profile-dropdown.tsx`, `nav-user.tsx`, `sidebar-data.ts` user block: identity
     comes from the stored token's principal if an endpoint exposes it, else a neutral
     "Operator" with a Sign-out that clears the token. No fictional names/emails.
   - `features/settings/` pages: keep appearance/display (they drive the real
     theme/font/layout providers); delete account/notifications forms if they submit nowhere.
   - `components/coming-soon.tsx`, unused assets (`clerk-logo`, brand icons), `cz.yaml`,
     `netlify.toml`, knip leftovers: run `pnpm knip`, delete what it flags (review each hit).
2. **Auth header**: `api/client.ts` attaches `Authorization: Bearer <token>` from the auth store
   when present; 401 → clear token, toast, redirect to sign-in (the template's QueryCache
   handler in `main.tsx` already sketches this — make it real, axios references removed).
3. **Tests**: the template's playwright browser suite is half-stale (asserts deleted routes).
   Either fix the survivors to the Rubix nav or delete them per-file with a one-line reason in
   the session log; the vitest unit suite (`*.unit.test.*`) is the canonical fast gate — extend
   it to cover: client auth header, 401 handling, one render test per major page against mocked
   fetch (test fixtures allowed).
4. **Gates made permanent**: add `pnpm -C rubix/ui lint` to the DONE GATE of any future UI row
   (fix what it finds now); add a `check:fake` script wrapping the grep gate so CI/loop runs it
   verbatim; verify FILE-LAYOUT discipline (`find src -name '*.ts*' | xargs wc -l | sort -rn |
   head` — nothing over 400 outside `components/ui/` and `routeTree.gen.ts`).
5. **Final look-freeze pass**: full screenshot sweep vs `ui/docs/reference/`; update README
   (dev loop, token setup, seed, scripts).

## Done means

`grep` gates clean, lint clean, unit tests green, no template fiction anywhere, bearer-token auth
works against the real server, and the final screenshots match the frozen look.
